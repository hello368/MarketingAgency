## 3. 기술 스택 및 시스템 아키텍처

```
Google Chat (Spaces)
       │
       │ HTTP POST (webhook)
       ▼
ngrok / Cloud Run
       │
       ▼
FastAPI (Python 3.12)  ←──────────────────────────────────┐
  │                                                        │
  ├── BackgroundTasks (즉시 200 반환, 무거운 처리는 비동기)   │
  │                                                        │
  ├── SQLite / db.py          (체크인·SLA 타이머 영속)      │
  ├── Google Sheets           (5탭 마스터 대시보드)         │
  │   ├── gsheet_handler.py   (Live Status 전담)           │
  │   └── sheets_client.py    (나머지 4탭, threading.Lock) │
  ├── Telegram Bot            (@Michael_Agency_Bot)        │
  │   └── telegram_bot.py                                  │
  └── OpenRouter AI           (model_router.py)  ──────────┘
      ├── anthropic/claude-sonnet-4.6   [high_intelligence]
      ├── google/gemini-2.5-pro         [long_context]
      └── deepseek/deepseek-chat        [fast_reply]

공유 상태:
  state.py  →  gsheet 인스턴스를 main.py가 startup에서 설정
               scheduler.py가 순환 임포트 없이 읽음
```

| 레이어 | 기술 | 비고 |
|---|---|---|
| Web Framework | FastAPI + uvicorn | BackgroundTasks로 즉시 200 반환 |
| DB | SQLite | `/data/tracking.db` |
| Google 연동 | gspread + google-auth | Service Account 인증 |
| Telegram | pyTelegramBotAPI | `/register [이름]` 커맨드 |
| AI | OpenRouter (model_router.py) | 3-tier 자동 라우팅 + Fallback |
| 스케줄러 | APScheduler | 4 cron 잡 + 1 interval 잡 |
| 터널 | ngrok (임시) → Cloud Run (목표) | Space ID 설정 완료 |

## 5. 기능 명세

### 5-1. 스케줄러 잡 (4 cron + 1 interval)

| 시간 (Manila, UTC+8) | 유형 | 잡 ID | 동작 |
|---|---|---|---|
| 08:55 | cron | checkin_prompt | Daily Report 채널에 체크인 프롬프트 발송, 스레드 키 저장 |
| 09:16 | cron | late_sweep | 미체크인 멤버 → Sheets 기록 + Michael Telegram 알림 (AI 문구) |
| 16:45 | cron | eod_prompt | EOD 제출 프롬프트 발송, 스레드 키 저장 |
| 17:01 | cron | eod_sweep | 미제출 멤버 → Sheets 기록 + Michael Telegram 알림 (AI 문구) |
| 매 15초 | interval | sla_check | 만료 SLA 타이머 확인 → 해당 멤버 Telegram 알림 발사 (AI 문구) |

> **sla_check 폴링 주기:** SQLite 동시 접근 경합을 고려해 5초 → **15초**로 조정 완료.
> SLA_SECONDS=900(프로덕션) 기준 최대 지연 15초 → 운영상 무관함.

### 5-2. EOD 시간 윈도우 (수정 완료)

| 구분 | 값 |
|---|---|
| 프롬프트 발송 | 16:45 (스레드 키 생성 시점) |
| 윈도우 시작 | **16:45** ← 수정 (기존 16:30은 스레드 키 미존재 구간) |
| 윈도우 종료 | 17:00 |
| EOD Sweep | 17:01 |

### 5-3. Google Sheets 5탭 구조

| 탭 | 주요 컬럼 |
|---|---|
| **Live Status** | Name / Status (emoji) / Last Active / Telegram Pings / Notes |
| **Daily Check-ins** | Date / Name / Group / Time / Goal 1-3 / Status |
| **EOD Results** | Date / Name / Group / Submit Time / Link 1-3 / Status |
| **SLA Ping Log** | Date / Time / Space / Tagger / Tagged / Thread / 15-Min Met? / Pings |
| **Ping Summary** | Name / Group / Weekly / Monthly / Total / Last Breach |

### 5-4. AI ModelRouter 라우팅 기준

| Task Type | 감지 조건 | Primary | Fallback 1 | Fallback 2 |
|---|---|---|---|---|

| `fast_reply` | status/alert/check/ping 키워드 또는 기본값 | deepseek-chat | claude-sonnet-4.6 | gemini-2.5-pro |

**실제 사용 위치:** SLA 알림 문구, late sweep 보고, EOD sweep 보고
